Jpa

JPA是一种规范，需要由具体的实现，体现了面向接口的特性。

1.JPA是什么

  1. Java Persistence API：用于对象持久化的API
  2. Java EE 5.0 平台标准的ORM规范，使得应用程序徐以统一的方式访问持久层

2.JPA和Hibernate的关系

  1. JPA是Hibernate的一个抽象（就像JDBC和JDBC驱动的关系）
  2. JPA是规范：JPA本质上就是一种ORM规范，不是ORM框架。这因为JPA并未提供ORM实现，他只是制定了一些规范，提供了一些编程的API接口，但具体的实现是由ORM厂商实现的。
  3. Hibernate是实现：Hibernate除了作为ORM框架之外，它也是一种JPA实现
  4. 从功能上来说，JPA是Hibernate功能的子集

3.JPA的供应商

JPA的目标之一是制定一个可以由很多供应商实现的API，Hibernate3.2+、TopLink 10.1+以及OpenJPA都提供了JPA的实现，JPA的供应商有很多，常见以下四种：

  1. Hibernate
    JPA的始作俑者就是Hibernate的作者，Hibernate从3.2开始兼容JPA
  2. OpenJPA
    OpenJPA是Apache组织提供的开源项目
  3. TopLink
  4. EclipseLink

4.JPA的优势

  1. 标准化：提供相同的API，这保证了基于JAP开发的企业应用能够经过少量的修改就能在不用的JPA框架下运行。
  2. 简单易容，集成方便：JPA的主要目标之一就是提供更加简单的编程模型，在JPA框架下创建实体和创建Java类一样简单，只需要使用 javax.persistence.Entity 进行注解；JPA的框架和接口也都非常简单。
  3. 可媲美JDBC的查询能力：JPA的查询语言是面向对象的，JPA定义了独特的JPQL，而且能够支持批量更新和修改、JOIN、GROUP BY、HAVING等通常只有SQL才能提供的高级查询特性，甚至还能支持子查询。
  4. 支持面向对象的高级特性：JPA中能够支持面向对象的高级特性，如类之间的复杂关系、最大限度的使用面向对象的模型。

5.Spring Data

Spring Data是Spring的一个子项目。用于简化数据库访问，支持NoSQL和关系型数据库。主要目标是使数据库的访问变得快捷。Spring Data具有如下特点：

1.SpringData项目支持NoSQL存储：

​ MongoDB（文档数据库）

​ Neo4j（图形数据库）

​ Redis（键/值存储）

​ Hbase（列族数据库）

2.SpringData项目所支持的关系型数据存储技术

​ JDBC

​ JPA
