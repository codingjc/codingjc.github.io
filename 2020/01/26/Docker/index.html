Docker

1、简介

Docker是一款开源的应用容器引擎，轻量级容器技术。性能远远高于虚拟机。

Docker支持将软件编译成一个镜像；然后在镜像中各种软件做好配置，将镜像发布出去，其他使用者可以直接使用这个镜像。

运行中的这个镜像称为容器，容器的启动是毫秒级别的。

2、核心概念

  • 镜像(Image): Docker镜像是用于创建Docker容器的模板。
  • 容器(Container): 容器是独立运行的一个或一组应用。
  • 客户端(Client): 客户端通过命令行或其他工具来连接Server使用Docker API与Docker守护进程通信。
  • 主机(Host): 一个物理或虚拟的机器用于执行Docker守护进程和容器。
  • 仓库(Registry): Docker仓库是用来保存镜像，可以理解为代码控制的代码仓库。

使用Docker的步骤：

  1. 安装Docker
  2. Docker仓库找个这个软件对应的镜像
  3. 使用Docker运行这个镜像，这个镜像就会生成一个Docker容器
  4. 对容器的启动、停止就是对软件的启动、停止

3、安装Docker

本次安装是在Linex环境上安装

​ 1. 查看Linux内核的版本(Docker要求CentOS系统的内核版本高与3.10)

​ 2. 升级软件包以及内核

​ 3. 安装Docker

​ 4. 启动Docker

​ 5. 将Docker设置为开机自动启动

4、常用操作

4.1 镜像操作

镜像仓库地址: https://hub.docker.com/

操作 命令 说明
检索 docker search keywod 去docker hub上检索镜像信息
拉取 docker pull 镜像名:tag tag是可选的，指的是软件的版本，默认不指定为latest
列表 docker images 查看本地所有镜像
删除 docker rmi images-id 删除指定的镜像

4.2 容器操作

流程: 软件镜像 –> 运行容器 –> 产生一个容器(正在运行的软件)

操作 命令 说明
运行 docker run --name container-name -d image-name –name: 指定容器名；-d: 后台运行
列表 docker ps (查看运行中的容器) 后面加上-a，查看所有容器
停止 docker stop container-name/container-id 停止指定容器
启动 docker start container-name/container-id 启动指定容器
删除 docker rmi container-id 删除指定容器
端口映射 eg: docker run -d -p 8888:8080 –name mytomcat docker.io/tomcat -p : 主机端口:容器内部端口
容器日志 docker logs container-name/container-id 查看容器内部日志
